# English Code & Documentation Rule

## Scope
- **ALL code** (variable names, function names, comments, docstrings, error messages, CLI output) MUST be written in **English**.
- **ALL documentation** (README.md, docs/*.md, task specs, guides, commit notes in AIVC) MUST be written in **English**.
- **Chat & Artifacts** (brainstorming.md, walkthroughs, implementation plans, task.md, and direct conversation with the user) remain in the **user's language** (French by default).

## Summary
| Context | Language |
|---------|----------|
| Code (Python, JS, etc.) | 🇬🇧 English |
| In-code comments & docstrings | 🇬🇧 English |
| CLI output & error messages | 🇬🇧 English |
| Documentation (`docs/`, `README.md`) | 🇬🇧 English |
| AIVC commit titles & notes | 🇬🇧 English |
| Chat with user | 🇫🇷 User's language |
| Agent artifacts (plans, walkthroughs) | 🇫🇷 User's language |
